## Dual-formula shadow mode (decision C)

Python's live `model_registry.uvt(input, output, cached)` = `(input − cached) + output`. No weights, no model multipliers. It's the currency of `usage_events.uvt_counted` and `uvt_balances.total_uvt`. Changing it is out of scope for PR 1.

Spec's weighted formula with model multipliers is a DIFFERENT currency. PR 1 computes BOTH in PolicyGate:

- `computeUvtWeighted(usage)` → spec §2 formula, 7 inputs, v1 weights × model multiplier → logged as `routing_decisions.predicted_uvt_cost`
- `computeUvtSimple(usage)` → `(input - cached) + output` → logged as `routing_decisions.predicted_uvt_cost_simple`

**All gate arithmetic (balance check, opus_pct_mtd) uses SIMPLE.** Python-parity. `lib/router/constants.ts` exports `UVT_FORMULA_ENFORCEMENT = "simple"`. PR 2 flips it to `"weighted"` after backfill.

This means `InsufficientUvtBalanceError.observedValue` is the simple count, not the weighted. `opus_pct_mtd` sums `usage_events` where `model='opus'` in the simple unit.

## Model ID mapping (DB CHECK constraint)

`public.usage_events.model` is CHECK-constrained to `haiku|sonnet|opus|gpt5|gemma` (Python short keys). PolicyGate's `chosen_model` uses the spec's logical names (`claude-haiku-4`, `claude-sonnet-4`, `claude-opus-4`, `gpt-5`, `gpt-5-mini`, `perplexity-sonar`). `routing_decisions.chosen_model` is a NEW column with NO enum — stores logical names.

`site/lib/router/model_id_map.ts` translates logical → short for any caller that writes through the CHECK-constrained path:

```ts
LOGICAL_TO_SHORT = {
  "claude-haiku-4": "haiku",
  "claude-sonnet-4": "sonnet",
  "claude-opus-4": "opus",
  "gpt-5-mini": "gpt5",          // TEMP — no mini in Python today
  "gpt-5": "gpt5",
  "perplexity-sonar": "gemma",   // PLACEHOLDER — no sonar bridge yet
}
```

Snapshot IDs (`claude-haiku-4-5-20251001`, etc.) resolve inside Python `lib/model_registry.py`. PolicyGate never sees snapshots.
